# R5: SVG 无障碍语义注入（A11y）

## 文档信息

- **项目**: image-annotator-mcp
- **需求编号**: R5
- **所属阶段**: Phase 2 / P2
- **创建日期**: 2026-03-16
- **状态**: 待开发（依赖 SVG 输出格式支持）
- **估计工作量**: S（1 ~ 2 天）

---

## 1. 问题描述

当前系统将所有标注输出为 PNG/WebP/AVIF 等光栅化格式，SVG 仅作为内部中间层在 `sharp.composite()` 后被丢弃。

Phase 1 已实现基于元数据的 `alt_text` 字段（在 MCP 响应中），但这只是纯文本描述，对以下场景无效：

- 页面内联 SVG（直接渲染在 HTML 中）
- 屏幕阅读器（JAWS / NVDA）需要解析 SVG 语义
- WCAG 2.1 合规检查

当输出格式为 SVG 时，视障用户使用辅助技术无法读出任何有意义的标注内容。

---

## 2. 前提条件

本需求依赖 **先将 `svg` 加入 `output_format` 枚举**，因为当前系统不支持 SVG 直接输出。

该前提条件本身的工作量较小，可作为本需求的子任务一并实施。

---

## 3. 方案设计

### 3.1 支持 SVG 格式输出

在 `output_format` 枚举中加入 `svg`：

```javascript
// server.js inputSchema
enum: ['png', 'jpeg', 'webp', 'avif', 'svg']
```

在 `annotate.js` 的 `annotateImage()` 中，当 `outputFormat === 'svg'` 时，跳过 sharp compositing，直接将 SVG 字符串写入输出文件：

```javascript
if (outputFormat === 'svg') {
  fs.writeFileSync(finalOutputPath, optimizedSvg, 'utf-8');
} else {
  // 原有 sharp 合成路径
  let pipeline = sharp(inputPath).composite([...]);
  // ...
}
```

注意：SVG 输出不含原始截图（SVG 是纯标注层，不是合成结果）。若需要包含底图，可将底图嵌入 SVG 的 `<image>` 标签中（可选扩展）。

### 3.2 ARIA 语义注入

在 `buildSvg()` 中，为 SVG 注入无障碍属性：

#### 3.2.1 根节点

```xml
<svg
  width="1920" height="1080"
  xmlns="http://www.w3.org/2000/svg"
  role="img"
  aria-labelledby="svg-title-{id} svg-desc-{id}">
  <title id="svg-title-{id}">{简短标题}</title>
  <desc id="svg-desc-{id}">{generateAltText() 的完整描述}</desc>
  <defs>...</defs>
  ...
</svg>
```

`{id}` 使用 `generateId('svg')` 保证唯一性。

标题从标注上下文推断：
- 有 theme → `"{theme} 标注图 ({width}×{height})"`
- 无 theme → `"标注图 ({width}×{height})，共 {N} 个标注"`

#### 3.2.2 标注组节点

每个 `<g>` 包裹单个 annotation 的所有 SVG 元素，并添加 `aria-label`：

```xml
<g aria-label="第 1 个标注：标记圆，位置 (100, 200)，编号 1">
  <!-- createMarker 返回的所有 svg 元素 -->
</g>
```

不同类型的 `aria-label` 模板：

| 类型 | 模板 |
|------|------|
| `marker` | `"编号标记 #N，位置 (x, y)"` |
| `arrow` | `"箭头，从 (x1,y1) 指向 (x2,y2)"` |
| `callout` | `"说明气泡，位置 (x,y)，内容：{text}"` |
| `label` | `"文字标签，位置 (x,y)，内容：{text}"` |
| `blur` | `"模糊遮蔽区域，覆盖 (x,y,w,h)"` |
| `highlight` | `"高亮区域，覆盖 (x,y,w,h)"` |
| `rect` | `"矩形标注，覆盖 (x,y,w,h)"` |
| `circle` | `"圆形标注，圆心 (x,y)，半径 r"` |
| `icon` | `"图标 {icon}，位置 (x,y)"` |
| `connector` | `"连接线，从 (x1,y1) 到 (x2,y2)"` |
| `curved-arrow` | `"曲线箭头，从 (x1,y1) 指向 (x2,y2)"` |

#### 3.2.3 实现位置

在 `buildSvg()` 中：

1. 将每个 `create*()` 的返回值包装进 `<g aria-label="...">...</g>`
2. 在 SVG 根节点注入 `role` / `aria-labelledby`
3. 在 `<defs>` 前插入 `<title>` 和 `<desc>`

## 5. 验收标准

- [ ] `output_format: 'svg'` 时，工具直接输出 SVG 文件
- [ ] SVG 根节点包含 `role="img"` 和 `aria-labelledby`
- [ ] SVG 包含 `<title>` 和 `<desc>` 标签，内容来自 `generateAltText()`
- [ ] 每个标注 `<g>` 元素包含 `aria-label` 描述
- [ ] `npm test` 全部通过
- [ ] W3C SVG 无障碍验证工具可识别 title 和 desc

---

## 6. 注意事项

- SVG 输出不包含底图像素数据（只有标注层），适用于叠加渲染场景
- 若需要包含底图，可在 SVG 中嵌入 `<image href="data:image/png;base64,..."/>`，但会显著增大文件体积（可选扩展）
- ARIA 注入对 PNG/WebP/AVIF 光栅化输出无效，这些格式依然只能依靠 `alt_text` 字段
